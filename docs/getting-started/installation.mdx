Build from source for development; grab a release binary or the container image to just run it. For source builds, tooling (Rust, cmake, pkg-config) is managed with [mise](https://mise.jdx.dev); OpenSSL can come from any source (nix, Homebrew, …) as long as `pkg-config` can resolve it.

<Steps>
  <Step title="Clone and set up the toolchain">
    ```bash
    git clone https://github.com/tylerdavis/graph
    cd graph
    mise trust
    mise install
    ```
  </Step>
  <Step title="Build and install">
    ```bash
    mise run install
    ```
    This builds the release binary and installs `graph` onto your PATH (`~/.cargo/bin`). Plain `cargo build` works too — the embedded database's linker requirements are handled by build scripts in the workspace.
  </Step>
  <Step title="Verify">
    ```bash
    graph --help
    ```
    <Info>
    The very first run of a freshly built binary pauses ~1s while macOS scans it (Gatekeeper). Every run after that is instant, until the next rebuild.
    </Info>
  </Step>
</Steps>

## Release binaries

Tagged releases publish binaries for macOS (arm64) and Linux (x86_64) with SHA-256 checksums:

```bash
curl -fsSLO "https://github.com/tylerdavis/graph/releases/download/v0.3.0/graph-v0.3.0-x86_64-unknown-linux-gnu.tar.gz"
tar xzf graph-v0.3.0-x86_64-unknown-linux-gnu.tar.gz && install -m 0755 graph ~/.local/bin/
```

## Container image

Each release also publishes `ghcr.io/tylerdavis/graph` (tags `vX.Y.Z`, `X.Y`, `latest`): the Linux binary plus `git`, `jq`, and `gh` — everything the [github tool pack](/tools/user-defined#bundled-packs) shells out to. It doubles as the runtime for [CI jobs](/cookbook/ci-checks) (no install step) and works standalone:

```bash
docker run --rm -e ANTHROPIC_API_KEY -e GRAPH_STORAGE=memory \
  -v "$PWD:/work" -w /work ghcr.io/tylerdavis/graph:latest plan list
```

While the repository is private the image is too — `docker login ghcr.io` with a token that has `read:packages` first (CI jobs use the workflow's `GITHUB_TOKEN`).

## Requirements

| Requirement | Notes |
|---|---|
| Rust 1.94+ | installed by `mise install` |
| cmake, pkg-config | installed by `mise install` |
| OpenSSL | from nix, Homebrew, or system — must be visible to `pkg-config` |
| An LLM provider key | e.g. `ANTHROPIC_API_KEY` — see [Quickstart](/getting-started/quickstart) |

## First-run notes

- graph creates its embedded database at `~/.local/share/graph/db/` on first use (configurable via `[settings] data_dir`).
- The embedded database is **single-process**: one running `graph chat` holds a lock that blocks other graph commands against the same data directory. Use `GRAPH_STORAGE=memory` for concurrent/ephemeral runs — see [Storage](/architecture/storage).
