Or from the command line:

<Tabs>
  <Tab title="macOS (arm64)">
    ```bash
    curl -fsSLO "https://github.com/tylerdavis/graph/releases/download/{{release_version}}/graph-{{release_version}}-macos-arm64.tar.gz"
    tar xzf graph-*.tar.gz && install -m 0755 graph ~/.local/bin/
    graph --help
    ```
  </Tab>
  <Tab title="Linux (x86_64)">
    ```bash
    curl -fsSLO "https://github.com/tylerdavis/graph/releases/download/{{release_version}}/graph-{{release_version}}-linux-x86_64.tar.gz"
    tar xzf graph-*.tar.gz && install -m 0755 graph ~/.local/bin/
    graph --help
    ```
  </Tab>
</Tabs>

<Info>
On macOS, the first run of a freshly downloaded binary pauses ~1s while Gatekeeper scans it. Every run after that is instant.
</Info>

## Container image

Each release also publishes `ghcr.io/tylerdavis/graph` (tags `vX.Y.Z`, `X.Y`, `latest`): the Linux binary plus `git`, `jq`, and `gh` — everything the [github tool pack](/tools/builtins) shells out to. It doubles as the runtime for [CI jobs](/cookbook/ci-checks) (no install step) and works standalone:

```bash
docker run --rm -e ANTHROPIC_API_KEY -e GRAPH_STORAGE=memory \
  -v "$PWD:/work" -w /work ghcr.io/tylerdavis/graph:{{release_version}} plan list
```

While the repository is private the image is too — `docker login ghcr.io` with a token that has `read:packages` first (CI jobs use the workflow's `GITHUB_TOKEN`).

## Requirements

| Requirement | Notes |
|---|---|
| macOS (arm64) or Linux (x86_64) | the platforms release binaries are published for |
| An LLM provider key | e.g. `ANTHROPIC_API_KEY` — see [Quickstart](/getting-started/quickstart) |

## First-run notes

- graph creates its data directory at `~/.local/share/graph/` on first use (configurable via `[settings] data_dir`).
- State is plain files under the data directory, safe under concurrent runs. Use `GRAPH_STORAGE=memory` for ephemeral runs that should leave nothing behind — see [Storage](/architecture/storage).
